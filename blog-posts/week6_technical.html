Blocks, Procs, and Lambdas

Blocks

Many Ruby methods use blocks to pass optional arguments to the method. Code blocks use either a set of {} or a do..end construct. Many blocks in ruby also take something called “block arguments”. If you’ve ever seen a code block that looks like this { |x| puts x }, the part inside the || is the block argument. It can have more than one variable, they just need to be separated by commas. Blocks in Ruby, break the Ruby rule of most everything is an object, since blocks are not objects.

Procs

A proc in Ruby, is a saved block that can be used over and over again. You define a proc with Proc.new syntax. Any method that would normally take a code block can be passed a proc instead. To use a proc, you (&name_of_proc) syntax to call it. Unlike code blocks, procs are ruby objects. The usefulness of procs is that it saves you from writing the same blocks of code over and over again.

Lambdas

Lambdas are Ruby objects, like most all of the Ruby language. A lambda is just like a proc, only it cares about the number of arguments it gets and returns to its calling method rather than returning immediately. A lambda is just like a regular method, except it has no name. You can assign them to variables and also pass them parameters inside of ||. Lambdas work with both the {} and the do..end syntax.
