Technical Week 7


SQL Injection

What is SQL Injection?

Is a technique used to attack data driven web applications. It exploits security vulnerabilities in the application being attacked. A SQL injection attack consists of injecting a SQL query from client side input data fields, allowing the attacker to read sensitive data and or modify (create, insert, update, delete) the data on the database.

Why is SQL Injection a Problem?

SQL Injection attacks can completely destroy your database of users or have a breach in the security and give the attacker all of your users information, passwords, usernames, address’s, credit card numbers the list can go on and on. We all have received emails saying that some company’s system has been breached and we should go and change our passwords immediately. This is most likely due to an SQL attack. People can also use SQL injections to get around user authentication. 0’or’0’=’0 is a popular SQL injection to bypass passwords. This is also used to bypass the username field sometimes as well.

What Can Programmers Do?

The most common solution to minimize SQL injections involves running any submitted data through a function to ensure that any dangerous characters like " ' " are not passed to a SQL query in data.
